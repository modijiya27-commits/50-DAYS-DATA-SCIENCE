# Day 3/50 — Spotify Tracks EDA  
## Series: Not A Data Scientist — Yet

This project is part of my **50 Days of "Not A Data Scientist — Yet"** learning series.

On Day 3, I worked on **Exploratory Data Analysis (EDA)** using a Spotify Tracks dataset.  
The focus was on understanding the dataset, cleaning basic issues, analyzing popularity trends, and visualizing relationships between different audio features.

---

## Project Objective

The main objective of this project was to practice EDA by exploring a music dataset and answering questions such as:

- Which artists have the highest average popularity?
- Which genres are most popular?
- How is track popularity distributed?
- Does song duration affect popularity?
- Is danceability related to popularity?
- Are explicit songs more popular than non-explicit songs?
- What relationships exist between numerical audio features?

---

## Dataset

The dataset contains Spotify track-related information such as:

- Track name
- Artist name
- Genre
- Popularity
- Duration
- Danceability
- Explicit content
- Other numerical audio features

---

## Tools and Libraries Used

- Python
- Pandas
- Matplotlib
- Seaborn
- Jupyter Notebook

---

## Steps Performed

### 1. Data Loading

Loaded the dataset into a Pandas DataFrame and inspected the basic structure.

Checked:

- Dataset shape
- Column names
- Data types
- First few rows

---

### 2. Data Cleaning

Performed basic cleaning steps including:

- Checking missing values
- Handling null values
- Converting the `popularity` column into numeric format
- Preparing the dataset for analysis and visualization

---

### 3. Exploratory Data Analysis

Performed different EDA tasks to understand the dataset better.

Analysis included:

- Top artists by average popularity
- Top genres by average popularity
- Distribution of track popularity
- Duration vs popularity analysis
- Danceability vs popularity analysis
- Explicit vs non-explicit track popularity
- Correlation between numerical features

---

## Visualizations Created

The notebook includes visualizations such as:

- Bar chart of top artists by average popularity
- Bar chart of top genres by average popularity
- Popularity distribution plot
- Scatter plot of duration vs popularity
- Scatter plot of danceability vs popularity
- Comparison of explicit and non-explicit tracks
- Correlation heatmap

---

## Key Learnings

From this project, I learned that:

- EDA is not only about creating charts, but about asking useful questions from data.
- Cleaning data is an important step before analysis.
- Visualizations help reveal patterns that are not obvious from raw tables.
- Popularity can be analyzed from multiple angles such as artist, genre, duration, and audio features.
- Correlation heatmaps are useful for quickly understanding relationships between numerical variables.
